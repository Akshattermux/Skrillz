## ✨ Key Features

### 🩺 Consultant & Telehealth System
* **Intelligent Onboarding:** A slick, interactive intro UI directing users smoothly into the ecosystem.
* **Dynamic Dashboard:** Personalized greeting, banner highlights, and instant search functions for specialized clinical fields.
* **Advanced Discovery System:** Live search indexing alongside dedicated medical category groupings (e.g., General, Cardiology, Dental, Neurology).
* **Rich Specialist Profiles:** Detailed views showcasing experience metrics, patient ratings, bio statements, and a robust date/time slot booking manager.
* **Comprehensive Appointment Logs:** Dedicated schedules view to monitor upcoming, completed, or cancelled medical consultations.

### 🎓 Learning & Skilling Ecosystem
* **Engaging Course Discovery:** A dedicated educational frontend showcasing running tech cohorts, progress charts, and tailored search filtering.
* **Interactive Learning Player:** Multi-tab layout tracking course syllabus checkpoints, lesson resources, video components, and student reviews.
* **Native Account Architecture:** Structured profile menus linking account preferences, app security, notification settings, and easy logout loops.

---

## 🛠️ Tech Stack & Architecture

* **Framework:** [React Native](https://reactnative.dev/) with [Expo](https://expo.dev/) (Managed Workflow)
* **Routing System:** `expo-router` (File-based safe native navigation)
* **Styling Engine:** Native `StyleSheet` utilizing flexible grid architectures
* **Performance Assets:** `expo-image` integration for lightning-fast asset rendering and memory-cached graphics optimization

## 🚀 Getting Started

Follow these instructions to spin up the development environment locally.

### Prerequisites
Make sure you have Node.js installed on your development machine. Then, install the Expo Go client app on your physical iOS or Android testing device.

### Installation & Run Steps

1. **Clone the Repository:**
   ```bash
   git clone [https://github.com/Akshattermux/Skrillz.git](https://github.com/Akshattermux/Skrillz.git)
   cd Skrillz
## Get started

1. Install dependencies

   ```bash
   npm install
   ```

2. Start the app

   ```bash
   npx expo start
   ```

In the output, you'll find options to open the app in a

- [development build](https://docs.expo.dev/develop/development-builds/introduction/)
- [Android emulator](https://docs.expo.dev/workflow/android-studio-emulator/)
- [iOS simulator](https://docs.expo.dev/workflow/ios-simulator/)
- [Expo Go](https://expo.dev/go), a limited sandbox for trying out app development with Expo

You can start developing by editing the files inside the **app** directory. This project uses [file-based routing](https://docs.expo.dev/router/introduction).

## Get a fresh project

When you're ready, run:

```bash
npm run reset-project
```

This command will move the starter code to the **app-example** directory and create a blank **app** directory where you can start developing.
